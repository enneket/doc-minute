## 1. 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React + Vite |
| 后端 | Node.js + Express |
| 数据库 | SQLite (better-sqlite3) |

---

## 2. 数据模型

### notes 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| title | TEXT | 纪要标题 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### items 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| note_id | INTEGER | 外键，关联 notes.id |
| content | TEXT | 条目内容 |
| completed | BOOLEAN | 是否完成，默认 false |
| order_index | INTEGER | 排序索引 |
| created_at | DATETIME | 创建时间 |

---

## 3. API 设计

| Method | Endpoint | 说明 |
|--------|----------|------|
| GET | /api/notes | 获取所有纪要列表 |
| POST | /api/notes | 创建新纪要 |
| DELETE | /api/notes/:id | 删除纪要 |
| GET | /api/notes/:id/items | 获取纪要的所有条目 |
| POST | /api/notes/:id/items | 添加条目 |
| PATCH | /api/items/:id | 更新条目（内容/完成状态） |
| DELETE | /api/items/:id | 删除条目 |

---

## 4. 前端交互流程

1. 首页展示纪要列表（标题 + 创建时间 + 完成进度）
2. 点击右上角 **+** 按钮 → 弹出模态框
3. 模态框内输入纪要标题，底部逐条添加内容
4. 每条内容左侧有 **☐** 复选框，点击标记完成
5. 点击保存 → 关闭弹框 → 首页更新
